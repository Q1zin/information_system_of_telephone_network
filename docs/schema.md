# Модель данных ГТС

## Сущности

### Геолокация и АТС
- **city** — города для межгорода; ровно один помечен `is_home` (частичный уникальный индекс).
- **address** — адрес (индекс, район, улица, дом, квартира). Уникальность по полному набору.
- **pbx** — базовая АТС: имя, код, тип, район, техвозможность (`capacity_numbers`,
  `total_channels`, `free_channels`, `has_free_cable`).
- **pbx_city / pbx_department / pbx_institution** — подтипы (class-table inheritance,
  1:1 с `pbx`). Триггер `trg_pbx_subtype_check` держит `pbx.pbx_type` согласованным с подтипом.

### Номера и абоненты
- **phone_number** — номер: АТС, тип линии (`main/parallel/paired`), межгород
  (`none/open/closed`), статус (`free/reserved/active/blocked`), адрес установки.
- **subscriber** — абонент (ФИО, пол, дата рождения, категория простой/льготный,
  тип льготы, статус). Ссылается на номер (многие-к-одному) и собственный адрес.

### Звонки
- **call_record** — CDR: исходящий номер, тип (`local/internal/external/intercity`),
  город назначения (для межгорода), длительность, стоимость.

### Биллинг
- **tariff** — абонплата по (тип линии × наличие межгорода).
- **billing_settings** — однострочная конфигурация (скидка льготникам, пеня,
  день оплаты = 20, отсрочка после уведомления).
- **invoice / payment / penalty / notification** — счета, оплаты, пени, письменные уведомления.

### Прочее
- **installation_queue** — очередь на установку (обычная/льготная) + техвозможность.
- **public_phone** — общественные телефоны и таксофоны.

### Аутентификация и RBAC
- **app_user, role, permission, role_permission, user_role** — пользователи и
  настраиваемая ролевая модель. Права в формате `entity:action`. Суперадмин
  настраивает роли без изменения кода.

## Целостность на уровне БД (req. 1)

| Триггер / констрейнт | Правило |
|----------------------|---------|
| `trg_pbx_subtype_check` | подтип АТС соответствует `pbx_type` |
| `trg_number_intercity_check` | межгород только у городских АТС; у замкнутых сетей — `none` |
| `trg_subscriber_count_check` | main ≤ 1 абонент, paired ≤ 2, parallel — много |
| `trg_subscriber_same_house_check` | параллельные/спаренные абоненты — в одном доме |
| `trg_number_status_sync` | статус номера (free/active) синхронизируется с числом абонентов |
| `chk_privilege` | у льготника указан тип льготы, у простого — нет |
| `chk_pbx_channels`, `chk_intercity_city`, `chk_notice_deadline`, … | CHECK-ограничения |

## Представления и функции
- **fn_subscriber_monthly_fee(id)** — расчёт абонплаты (тариф × 50% для льготника).
- **v_subscriber_full** — абонент + номер + АТС + адрес + возраст.
- **v_subscriber_debt** — долг по абонплате / межгороду / пеням + дата старейшего долга.
- **v_pbx_stats** — ёмкость, свободные номера, число абонентов по АТС.

## Привязка к 13 запросам варианта

| # | Запрос | Основные таблицы/представления |
|---|--------|-------------------------------|
| 1 | Абоненты АТС (все/льготники/по возрасту/по фамилии) | `v_subscriber_full` |
| 2 | Свободные номера (АТС/ГТС/по району, техвозможность) | `phone_number.status='free'`, `v_pbx_stats` |
| 3 | Должники (АТС/ГТС/район/срок/тип долга/размер) | `v_subscriber_debt` + `v_subscriber_full` |
| 4 | АТС с макс/мин числом должников и суммой долга | `v_subscriber_debt` GROUP BY АТС |
| 5 | Общественные телефоны и таксофоны | `public_phone` |
| 6 | % простых/льготных (АТС/ГТС/район/тип АТС) | `subscriber.category` агрегаты |
| 7 | Абоненты с параллельными телефонами | `phone_number.line_type='parallel'` |
| 8 | Телефоны по адресу/дому/улице (+ межгород) | `phone_number` ⋈ `address` |
| 9 | Город с наибольшим числом межгорода | `call_record` ⋈ `city` |
| 10 | Полная информация по номеру | `v_subscriber_full` WHERE number=? |
| 11 | Спаренные, которые можно расспарить (есть свободный номер) | `line_type='paired'` + `v_pbx_stats.free_numbers` |
| 12 | Внутренние номера замкнутых АТС с < N внешних звонков за период | `call_record.call_type='external'` |
| 13 | Должники для уведомления/отключения/блокировки межгорода | `v_subscriber_debt` + `notification` + `billing_settings` |
